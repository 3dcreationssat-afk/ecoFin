# Assumptions

- The approved Base44 screenshots define visual direction, not validated calculations.
- Phase 1 may use synthetic demonstration values when clearly labeled.
- The household currency is USD for this release.
- Local persistence means SQLite for household/domain records and browser local storage for lightweight UI preferences only.
- Account, category, goal, contribution, and transaction-normalization workflows are Phase 1.5 SQLite-backed foundations, not CSV import workflows.
- Budget, debt payoff, cash-flow, safe-to-save, report export, and scenario outputs remain demonstration-only where validated engines are not implemented.
- Recurring expense detection is a local deterministic pattern engine and not a bill-management, cancellation, bank-connectivity, or forecasting service.
- Supplemental collapsible navigation references under `design/enhancements/collapsible-navigation/` are treated as approved shell enhancement guidance.
